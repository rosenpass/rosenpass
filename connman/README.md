# ConnMan integration for Rosenpass + WireGuard

This directory provides an integration workflow for using ConnMan-managed
WireGuard tunnels with Rosenpass-managed PSK rotation.

## What this provides

- `rosenpass-connman@.service`: systemd template unit for running
  `rosenpass exchange-config` per ConnMan profile.
- `setup-rosenpass-connman.sh`: generates local WireGuard keys, Rosenpass keys,
  and writes matching ConnMan and Rosenpass configs.
- `validate-config.sh`: validates that both configs are consistent and rejects
  static `WireGuard.PresharedKey` entries.
- `examples/`: minimal two-peer example configs.

## Architecture

ConnMan is responsible for creating and managing the WireGuard tunnel.
Rosenpass runs in parallel and injects rotating PSKs via the WireGuard broker
(`device` + `peer` in the Rosenpass TOML).

To avoid conflicting writers:

- ConnMan owns tunnel lifecycle, endpoint, addresses, routes, and DNS.
- Rosenpass owns only the preshared key.
- `WireGuard.PresharedKey` should not be set in ConnMan configs.

## Quick start

1. Install ConnMan WireGuard support and Rosenpass binaries.
2. Generate config for one peer:

```bash
sudo ./connman/setup-rosenpass-connman.sh \
  --instance peer-a \
  --provider-name "Peer A VPN" \
  --host 198.51.100.11 \
  --wg-address 10.44.0.1/24 \
  --allowed-ips 10.44.0.2/32 \
  --peer-wg-public-key xTIB9kXjA/J6fQ6QtrQ5r4W4jV6L7B0W9Y5f2SOdT0k= \
  --peer-rp-public-key-file /etc/rosenpass/connman/peer-a/peer-b-rp-public-key \
  --rp-endpoint 198.51.100.11:9999 \
  --rp-listen "[::]:9998" \
  --wg-endpoint-port 51820 \
  --wireguard-device wg0
```

3. Verify generated configs:

```bash
sudo ./connman/validate-config.sh --instance peer-a
```

4. Connect the ConnMan provider (for example via `connmanctl`) and start Rosenpass:

```bash
sudo systemctl daemon-reload
sudo cp connman/rosenpass-connman@.service /etc/systemd/system/
sudo systemctl restart connman-vpn.service
sudo systemctl enable --now rosenpass-connman@peer-a.service
```

5. Confirm PSK updates:

```bash
sudo wg show wg0 preshared-keys
```

## Notes

- ConnMan's WireGuard plugin chooses interface names (`wg0`, `wg1`, ...).
  Ensure `device` in the Rosenpass config matches the interface ConnMan created.
- The setup script writes private material to `/etc/rosenpass/connman/<instance>/`
  and a ConnMan provider config to `/var/lib/connman-vpn/<instance>.config`.
- Copy local public keys generated by setup script to the remote peer.

## Example files

- `examples/peer-a.config`, `examples/peer-b.config`: ConnMan provider files.
- `examples/peer-a-rosenpass.toml`, `examples/peer-b-rosenpass.toml`:
  Rosenpass configs for each side.
